# CAT-002 — Technology Catalog UI and Module Requirements

## 1. Experience Goal

The Technology Catalog should look like a mature internal standards library—not a generic card grid.

A user viewing the module should immediately understand that Bobkat IT has a deliberate, documented, repeatable technology operating model.

## 2. Catalog Landing Page

### Header

- Title: Technology Catalog
- Subtitle: Approved technologies that define the Bobkat IT Standard
- Summary metrics:
  - Preferred Technologies
  - Approved Products
  - Mapped StackScore Pillars
  - Available Playbooks
- Primary actions:
  - Add Technology
  - Manage Categories
  - Review Standards

### Featured Standard Section

Show the four core technologies as large, polished feature cards:

- Ubiquiti UniFi
- NinjaOne
- StackScore
- Uptime Kuma

Each card should display:

- Logo or icon
- Vendor
- Category
- Standard status badge
- One-sentence purpose
- Top capabilities
- Stack layer
- Number of mapped products
- Number of related playbooks
- Last reviewed date

### Catalog Library

Provide:

- Search
- Category filters
- Standard status filters
- Lifecycle filters
- Featured-only toggle
- Grid and table views

## 3. Technology Detail Page

### Overview Header

- Product logo or icon
- Technology name
- Vendor
- Product family
- Preferred/Approved badge
- Lifecycle badge
- Summary
- Last review date
- Edit action

### Recommended Tabs

1. Overview
2. Capabilities
3. Products
4. Deployment Standard
5. Business Value
6. StackScore Mapping
7. Playbooks
8. Governance

### Overview Tab

Include:

- Primary purpose
- Why Bobkat IT selected it
- Standard role
- Key business outcomes
- Related technologies
- Supported environment types

### Capabilities Tab

Use grouped capability cards with practical descriptions.

### Products Tab

Display real-world hardware models, editions, or service packages with:

- Product name
- Model or edition
- Best-fit environment
- Preferred indicator
- Lifecycle state
- Summary
- Important specifications

### Deployment Standard Tab

Show:

- Standard architecture
- Minimum deployment requirements
- Recommended deployment
- Optional enhancements
- Required documentation
- Post-deployment validation
- Ongoing management expectations

### StackScore Mapping Tab

Display:

- Related assessment pillars
- Relevant assessment questions
- Recommendation templates
- Risk areas addressed
- Projects that may be generated

### Playbooks Tab

Show implementation and operational playbooks, including status.

### Governance Tab

Show:

- Catalog owner
- Last review
- Next review
- Exception guidance
- Licensing notes
- Security considerations
- Change history

## 4. Visual Direction

The module should match StackScore's premium dark-blue visual style.

Use:

- Deep navy surfaces
- Layered cards
- Clear information hierarchy
- Soft borders
- Blue/cyan highlights
- Status badges
- Icon-led sections
- Minimal empty space
- Dense but readable detail views
- Responsive layout
- Collapsible navigation support

Avoid:

- Large empty cards
- Generic placeholder statistics
- Excessive gradients
- Marketing-style fluff
- Fake data that does not connect to the documented standard

## 5. Screenshot Readiness

The seeded module should be visually complete immediately after deployment.

The landing page should contain enough real content to support a polished product screenshot without requiring the administrator to create records manually.
