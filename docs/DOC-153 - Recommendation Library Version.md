Purpose

DOC-153 defines the standardized recommendation library used throughout StackScore.

Every recommendation generated by the application originates from this document.

The Recommendation Library provides a consistent, vendor-agnostic approach to technology improvement while ensuring reports, projects, planning, and technology maturity remain standardized.

This document contains recommendation definitions only.

Assessment logic is defined in DOC-152.

Scoring is defined in DOC-111.

Update DOC-000 after creation.

Do not generate application code.

--------------------------------------------------

Purpose of the Recommendation Library

Recommendations should not be dynamically invented.

Recommendations should be selected from a standardized library that provides consistent language, priorities, business outcomes, and implementation guidance.

This ensures every consultant delivers recommendations using the same methodology.

Recommendation Design Principles

Every recommendation must:

• Be vendor agnostic.
• Focus on business outcomes.
• Reduce technology risk or improve business capability.
• Strengthen one or more Technology Pillars.
• Be understandable to business leaders.
• Be reusable across clients.
• Support measurable maturity improvements.
