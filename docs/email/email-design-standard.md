# Email Design Standard

## Purpose

StackScore emails should feel like they come from a premium SaaS platform.

Every email should immediately communicate professionalism, trust, and clarity while matching the StackScore application.

---

# Design Goals

- Professional
- Modern
- Minimal
- Mobile responsive
- Easy to scan
- Action oriented

---

# Layout

Every email should follow this structure:

1. Header
2. Hero Title
3. Supporting Description
4. Primary Call-To-Action
5. Supporting Content
6. Security or Informational Notice
7. Footer

---

# Theme

Match the StackScore application.

Use:

- Dark blue branding
- White content cards
- Rounded corners
- Soft shadows
- Professional spacing
- Clean typography

Avoid:

- Plain text emails
- Generic corporate templates
- Excessive colors
- Marketing spam styling

---

# Primary CTA

Every email should contain one clear primary action.

Examples:

- Activate Account
- View Report
- Review Proposal
- Schedule Review
- Reset Password

Buttons should always be centered and visually prominent.

---

# Accessibility

Every email must:

- Include plain text fallback
- Support Outlook
- Support Gmail
- Support Apple Mail
- Be mobile responsive
- Use accessible button contrast
