# Purpose

The Recommendation Lifecycle Engine (RLE) governs the complete lifecycle of every recommendation generated by StackScore.

Recommendations are no longer static findings generated from an assessment. Instead, they become managed initiatives that progress through planning, approval, implementation, and completion while continuously updating the customer's Technology Maturity.

This engine serves as the operational bridge between:

- Assessments
- Technology Roadmaps
- Projects
- Technology Maturity
- Executive Reporting

---

# 1. Vision

Every recommendation should tell a story.

Instead of disappearing after an assessment, recommendations become living work items that evolve alongside the customer's technology environment.

Every recommendation should answer:

- What needs improvement?
- Why does it matter?
- What is its current status?
- Who owns it?
- When will it be completed?
- How much maturity has it added?

---

# 2. Lifecycle States

Every recommendation shall exist in exactly one lifecycle state.

| Status | Description |
|---------|-------------|
| Open | Automatically generated from assessment |
| Planned | Selected for future implementation |
| Quoted | Included in a proposal or Statement of Work |
| Approved | Customer approved implementation |
| In Progress | Work has begun |
| Completed | Successfully implemented |
| Deferred | Intentionally postponed |
| Rejected | Customer declined implementation |
| Not Applicable | Recommendation determined unnecessary |

---

# 3. Lifecycle Workflow

```text
Assessment Completed
        │
        ▼
      Open
        │
        ▼
     Planned
        │
        ▼
      Quoted
        │
        ▼
     Approved
        │
        ▼
    In Progress
        │
        ▼
    Completed
```

Alternative paths:

```text
Open
 │
 ├────────► Deferred
 │
 ├────────► Rejected
 │
 └────────► Not Applicable
```

Deferred recommendations may later return to Planned.

Rejected recommendations are considered closed.

Completed recommendations are permanent.

---

# 4. State Transition Rules

| From | Allowed To |
|------|------------|
| Open | Planned, Deferred, Rejected, Not Applicable |
| Planned | Quoted, Deferred |
| Quoted | Approved, Deferred |
| Approved | In Progress |
| In Progress | Completed, Deferred |
| Deferred | Planned |
| Completed | Closed (No further transitions) |
| Rejected | Closed |
| Not Applicable | Closed |

Illegal transitions shall be blocked.

# 7. Recommendation Status Effects

Each lifecycle state influences StackScore behavior.

| Status | Current Score | Planned Score | Dashboard |
|---------|:-------------:|:-------------:|:---------:|
| Open | ❌ | ❌ | Visible |
| Planned | ❌ | ✅ | Visible |
| Quoted | ❌ | ✅ | Visible |
| Approved | ❌ | ✅ | Visible |
| In Progress | ❌ | ✅ | Visible |
| Completed | ✅ | ✅ | Archived to Completed |
| Deferred | ❌ | ❌ | Deferred Section |
| Rejected | ❌ | ❌ | Hidden from Roadmap |
| Not Applicable | ❌ | ❌ | Hidden |

# 8. Dashboard Requirements

## Customer Dashboard

Customers shall see:

- Immediate Priorities
- Planned Initiatives
- In Progress
- Recently Completed
- Deferred Items
- Roadmap Progress

Customers shall **not** see:

- Internal consultant notes
- Sales pipeline
- Proposal details
- Internal estimates

---

## Consultant Dashboard

Consultants shall additionally see:

- Internal Notes
- Opportunity Value
- Estimated Labor
- Proposal Status
- Assigned Consultant
- Customer Communications
- Project Links

---

### Proposal Accepted

Automatically transition:

Quoted → Approved

---

### Project Started

Automatically transition:

In Progress → Completed

# 13. Design Philosophy

Recommendations are the operational heartbeat of StackScore.

Assessments establish the baseline.

Recommendations define the roadmap.

Projects execute the work.

Completed recommendations improve Technology Maturity.

The Recommendation Lifecycle Engine ensures every improvement made by a customer is measurable, traceable, and reflected throughout the platform, transforming StackScore from a one-time assessment tool into a continuous technology improvement system.
